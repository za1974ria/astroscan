## 3. PHASE 1 — FIX PARSER NOAA Kp

### 3.1 Découverte préalable
**`services/weather_service.py` était ABSENT du disque** (que `__pycache__/weather_service.cpython-312.pyc` existait). Les workers gunicorn fonctionnaient en bytecode loaded-in-memory. Restauré depuis le commit backup `86ba895` (qui l'avait capturé via `git add -A`).

### 3.2 Format NOAA confirmé live (mai 2026)
```json
[
  {"time_tag": "2026-04-27T00:00:00", "Kp": 3.0,  "a_running": 15, "station_count": 8},
  ...
  {"time_tag": "2026-05-04T15:00:00", "Kp": 4.0, "a_running": 27, "station_count": 8}
]
```
→ **list-of-dicts** avec clé `Kp` (capitale). L'ancien parser cherchait `latest[1]` (format list-of-lists obsolète) → toujours `None` → fallback Kp=0.0.

### 3.3 Patches appliqués
**`app/blueprints/weather/__init__.py`** lignes 108-122 :
```diff
-        if isinstance(latest, list) and len(latest) > 1:
-            raw_kp = latest[1]
+        if isinstance(latest, dict):
+            # Format actuel NOAA (mai 2026) : list of dicts
+            raw_kp = (
+                latest.get("Kp")
+                or latest.get("kp_index")
+                or latest.get("estimated_kp")
+            )
+        elif isinstance(latest, list) and len(latest) > 1:
+            # Legacy fallback (ancien format)
+            raw_kp = latest[1]
```

**`services/weather_service.py`** ligne 418-432 : même bloc, même transformation, dans `get_kp_index()`.

### 3.4 Validation isolée (Python direct)
```bash
python3 -c "from services.weather_service import get_kp_index; print(get_kp_index())"
# → (4.0, 'live')   ✅ kp=4.0 (live NOAA actuel), status='live'
```

### 3.5 Validation live endpoint (BLOQUÉE)
```
GET /api/aurore   →  kp=0.0 status=fallback   (workers anciens)
GET /api/aurores  →  kp=0.0 status=fallback   (workers anciens)
```
**Cause :** workers gunicorn ont chargé l'ancien bytecode au boot 19:03:14 UTC. Pour les recharger : `sudo systemctl reload astroscan` (ou `kill -HUP 3763746` en root).

## 4. PHASE 2 — DÉCOUPLAGE `/api/space-weather`

### 4.1 Schéma legacy détecté (frontend)
Templates et JS attendent : `kp_index`, `statut_magnetosphere`, `impact_orbital`, `mise_a_jour_utc`, `source`. Le `get_space_weather()` existant renvoyait un schéma différent (`kp`, `status`, `level`, `color`…), donc création d'un helper dédié.

### 4.2 Nouvelle fonction dans `services/weather_service.py`
```python
def get_space_weather_legacy():
    kp, status = get_kp_index()
    now_utc = datetime.now(timezone.utc).strftime("%Y-%m-%d %H:%M:%S")
    return {
        "mise_a_jour_utc": now_utc,
        "kp_index": kp,
        "statut_magnetosphere": _kp_status_legacy_fr(kp),
        "impact_orbital": _kp_impact_orbital_fr(kp),
        "source": "NOAA Space Weather Prediction Center" if status=="live"
                  else "NOAA (fallback)",
    }
```

### 4.3 Routes BP refactorées

- `/api/space-weather` : idem, partage la même clé de cache.
