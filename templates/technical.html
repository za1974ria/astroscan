Technical Metrics & Environment

Production-ready backend system with real-time orbital computation (SGP4) and resilient architecture.

Metrics

  • Average API latency: ~150-300 ms (estimated)
  • p95 latency: ~400-800 ms (estimated)
  • Cache hit rate: ~70-90% (estimated)
  • Uptime: >99% (service monitored via systemd)
  • API availability: 200 OK on critical routes (/api/iss, /api/sdr/passes)
  • Visitors: internal logs available (no public analytics yet)
  • SGP4 precision: real-time orbital propagation using NORAD TLE data
  • Estimated daily API calls: ~1k–5k (based on internal logs)
  • Satellite passes computed per day: several hundreds (depending on usage)

Technologies & Versions

  • Python 3.12
  • Flask (backend framework)
  • Gunicorn (WSGI server)
  • Nginx (reverse proxy)
  • sgp4 (orbital computation)
  • Jinja2 (templating)

Infrastructure

  • Hosting: Hetzner Cloud VPS
  • OS: Ubuntu 24.04 LTS
  • CPU/RAM: Standard VPS (Hetzner Cloud)
  • Deployment: manual systemd service
  • No containerization (direct Linux deployment)

Testing & Validation

  • Syntax validation: python -m py_compile
  • Smoke tests: curl on critical endpoints
  • Manual production validation after each deployment
  • No CI/CD pipeline yet (planned)

Security

  • HTTPS enabled (SSL certificate)
  • Reverse proxy isolation (Nginx -> Gunicorn)
  • Input handling via Flask
  • API timeout & fallback mechanisms
  • Rate limiting: planned
  • Secrets managed at server level (environment variables)

Roadmap

  • Multi-satellite tracking (beyond ISS)
  • Skyfield integration for advanced observation geometry
  • User authentication & personalized tracking
  • API monetization / SaaS model
  • Improved real-time dashboards and user-facing analytics
  • Integration of business-oriented features (usage tracking, API access tiers)

Designed to evolve from a technical platform into a production-ready data service.
