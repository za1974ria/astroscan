## 2. Top 5 faiblesses (impact ESA × facilité de correction)

### 🔴 #1 — Couverture du cœur scientifique quasi nulle
- **Problème** : un reviewer ESA regarde *en priorité* les modules de calcul. `hilal_compute.py` (212 stmts) → **3.8 %**. `control_tower/probes.py` (340 stmts) → **7.6 %**. `external_feeds.py` (211 stmts) → **9 %**. `sentinel_metrics.py` (177 stmts) → **0 %**.
- **Où** : `app/services/{hilal_compute,external_feeds,sentinel_metrics}.py`, `app/services/control_tower/{probes,classifiers}.py`.
- **Effort** : **M** — 1 jour par module pour des tests unitaires sur des vecteurs astro de référence (comparaison skyfield).

### 🔴 #2 — `app/` exclu de ruff, code applicatif non linté
- **Problème** : `pyproject.toml:[tool.ruff].include = ["tests/**/*.py"]` + `exclude = ["app", "services", "scripts", ...]`. La gate CI ruff sur `app/` reporte "No Python files found" → **All checks passed!**. Le badge "lint vert" est trompeur.
- **Où** : `pyproject.toml:[tool.ruff]`.
- **Effort** : **S** (étendre l'include) suivi de **M** (corriger les warnings qui sortiront). Risque : volume inconnu de warnings sur 30 460 LOC.

### 🟠 #3 — `station_web.py` (2 223 lignes) toujours actif via 71 imports
- **Problème** : la migration vers `app/__init__.py:create_app()` est **incomplète**. `app/bootstrap.py`, `app/hooks.py`, `app/__init__.py`, `app/services/lab_helpers.py` (et 16 autres fichiers) importent encore depuis l'ancien monolithe `from station_web import …`. Un reviewer qui ouvre la racine voit ce gros fichier orphelin et perd confiance.
- **Où** : `station_web.py` (racine) + 71 sites d'import dans `app/`.
- **Effort** : **L** — chaque import à migrer demande de localiser la définition d'origine dans station_web et de la déplacer. Plusieurs jours.

### 🟠 #4 — 124 tests skipped en CI (fixtures `.env`-dependent)
- **Problème** : sur 602 tests collectés, **20 %** ne tournent jamais. Cause : les fixtures `app` et `factory_app` `pytest.skip()` si `/root/astro_scan/.env` est inaccessible — ce qui est le cas en CI (où `SECRET_KEY` est injecté mais le fichier `.env` n'existe pas). 31 skips dans `test_legacy_routes`, 23 dans `test_legacy_critiques`, 16 dans `test_astrobrain_routes`, 13 dans `test_legacy_architecture`.
- **Où** : `tests/conftest.py:75-124` (les 2 fixtures), `tests/README.md:79`.
- **Effort** : **S** — refactorer les fixtures pour synthétiser une config minimale en `TESTING=1` (l'existant `_resolve_secret_key` gère déjà le fallback random). Gain immédiat : couverture mesurée ~25 % → ~35 %.

### 🟠 #5 — `CHANGELOG.md` figé à v2.0.0 (4 mai) vs dernier tag v2.7.4
- **Problème** : un reviewer ouvre `CHANGELOG.md`, voit "Mai 2026 — v2.0.0", et conclut que le projet est inactif depuis 3 semaines. La réalité : 7 versions intermédiaires (v2.7.0 → v2.7.4) jamais consignées. Le `README.md` affiche un badge "v2.8.0" qui n'existe même pas comme tag git.
- **Où** : `CHANGELOG.md`, `README.md` (ligne badge version).
- **Effort** : **S** — 2 h pour rédiger les entrées Keep-a-Changelog correspondant aux tags entre v2.0.0 et v2.7.4 (les messages de commit sont riches, c'est juste du formatage).
